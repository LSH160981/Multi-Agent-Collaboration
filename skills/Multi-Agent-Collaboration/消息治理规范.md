# 消息治理规范

## 面向用户的消息治理

只有主Agent可以向用户输出。

在输出前必须执行：

1. 去重：删除意思相同的多份表达
2. 合并：把多个异步回执合并成一条完整结论
3. 降噪：删除内部流程、无效中间状态、重复解释
4. 定稿：只保留最新有效结论
5. 复核：检查是否与历史已发送内容重复

## 其他 Agent 的输出要求

其他 Agent 不对用户说话，只输出给 Agent：

- 结构化 JSON
- 必要的 Markdown 说明
- 明确产物位置
- 明确验证方法
- 明确已知风险

## 禁止内容

- 禁止把内部讨论原样发给用户
- 禁止多个 Agent 各自向用户发重复结果
- 禁止把无意义中间状态作为“进展”频繁打扰用户
