# Multi-Agent-Collaboration 伪代码（中文，OpenClaw 原生多会话版）

> 核心前提：**Agent = OpenClaw session**。  
> 主Agent 负责唯一用户出口；其他 Agent 只进行内部协作，不得直接联系用户。

---

## 1. 总控流程伪代码

```text
函数 处理入口(用户输入):
    如果 用户输入 以 /mac 开头:
        入口模式 = "mac"
    否则如果 用户明确点名 Multi-Agent-Collaboration:
        入口模式 = "explicit-skill"
    否则如果 任务明显复杂:
        入口模式 = "default-takeover"
    否则:
        走普通对话流程
        结束

    主Agent = 当前主会话
    任务包 = 主Agent.解析任务(用户输入)

    如果 任务包.needs_clarification == true:
        主Agent.向用户提问(任务包.clarification_questions)
        结束

    控制层 = 初始化控制层会话()
        -> AgentPool
        -> 审核Agent
        -> 检查Agent

    编组方案 = AgentPool.生成编组方案(任务包)

    如果 编组方案.需要新招聘专家:
        AgentPool.创建或复用专家会话(Research / Verification / Summary / Coding / Ops ...)

    主Agent.下发任务到两组组长(任务包, 编组方案)

    并行执行:
        A组执行(稳定优先)
        B组执行(激进优先)
        检查Agent.持续巡检并在 stale 时恢复

    当 A组 和 B组 都有阶段性产出:
        审核报告A = 审核Agent.评分(A组结果)
        审核报告B = 审核Agent.评分(B组结果)

        如果 审核报告A.reject 且 审核报告B.reject:
            AgentPool.淘汰低效角色或整组
            检查Agent.生成故障分析与恢复建议
            返回 重新编组或重做阶段

        最优结果 = 审核Agent.择优(A组结果, B组结果)
        最终消息 = 主Agent.消息治理(最优结果, 历史用户消息)
        主Agent.向用户输出(最终消息)

    收尾:
        更新记忆
        更新日志
        更新任务队列
        git记录变更
```

## 2. 任务解析伪代码

```text
函数 解析任务(用户输入):
    去掉 /mac 前缀(如果存在)
    提炼 user_intent
    判断 task_type = research / coding / ops / mixed
    判断 complexity = low / medium / high
    判断 是否需要澄清
    生成 output_requirements
    固定 required_roles = [主Agent, AgentPool, 审核Agent, 检查Agent]

    如果 complexity >= medium:
        execution_mode = dual-group
    否则:
        execution_mode = single-group

    返回 标准任务包 JSON
```

## 3. 初始化控制层伪代码

```text
函数 初始化控制层会话():
    如果 已存在 main-ceo / pool-hr / review-judge / inspect-patrol:
        复用这些会话或其固定 agent 身份
    否则:
        创建控制层会话

    为每个控制层角色加载:
        abilities.md
        memory/
        logs/
        queue/

    返回 控制层句柄
```

## 4. 动态招聘伪代码

```text
函数 生成编组方案(任务包):
    读取 AgentPool 的能力清单
    分析任务所需能力

    基础角色固定:
        主Agent
        AgentPool
        审核Agent
        检查Agent

    若任务是 research:
        候选专家 = [Research, Verification, Summary]
    若任务是 coding:
        候选专家 = [Implementation, Test, Review]
    若任务是 mixed:
        候选专家 = [Research, Implementation, Verification, Summary]

    默认创建双组:
        A组 = 稳定优先
        B组 = 激进优先

    每组至少包含:
        组长 Lead
        必要专家若干

    优先复用:
        旧会话能力高、通过率高、最近健康 => 复用
    否则:
        创建新会话并写入角色边界

    返回 编组方案
```

## 5. Agent-to-Agent 派单伪代码

```text
函数 派单(发送方, 接收方, 任务包, 子目标):
    消息 = {
        type: "task_assign",
        task_id: 任务包.task_id,
        from: 发送方,
        to: 接收方,
        goal: 子目标.goal,
        inputs: 子目标.inputs,
        constraints: 子目标.constraints,
        deliverables: 子目标.deliverables,
        reply_to: 子目标.reply_to,
        status: "assigned"
    }

    写入发送方 logs/
    写入接收方 queue/
    通过 OpenClaw session/send 或 agent turn 投递
    接收方返回 task_ack
```

## 6. 并行竞争伪代码

```text
函数 双组并行执行(A组, B组):
    并发启动 A组 与 B组

    A组策略 = 稳定优先
        偏保守、重验证、重可复现

    B组策略 = 激进优先
        偏探索、重创新、重新路径

    两组都必须输出:
        summary
        verification
        risks
        artifacts

    结果统一交给审核Agent
```

## 7. 审核评分伪代码

```text
函数 审核并评分(组结果):
    Reviewer分 = 看格式完整性、输出是否齐全
    Judge分 = 看质量、推导、证据、逻辑
    Metrics分 = 看可验证性、稳定性、可执行性

    总分 = 加权(Reviewer分, Judge分, Metrics分)

    如果 缺少关键输出项:
        reject = true
    如果 存在编造、不可验证、明显跑题:
        reject = true
    如果 总分 < 阈值:
        reject = true

    返回 结构化评分卡
```

## 8. 巡检与恢复伪代码

```text
函数 巡检(Agent列表):
    每分钟执行一次:
        对每个 Agent:
            读取 queue/
            读取 logs/
            读取 最近产物
            读取 最后动作时间

            如果 超时无动作:
                标记 stale
                先发送 agent_ping
                再发送恢复指令

            如果 有日志但无产物:
                标记 watch

            如果 连续失败次数过多:
                检查是否需要替换会话

函数 恢复(Agent):
    如果 可重试:
        retry
    否则如果 可重派:
        派给同角色替代Agent
    否则:
        重建新Agent并继承必要上下文
```

## 9. 主Agent 消息治理伪代码

```text
函数 消息治理(候选结果, 历史消息):
    删除重复结论
    删除内部讨论噪音
    合并异步回执
    删除过期中间状态
    保留最新有效结论
    保留必要风险说明
    生成用户最终可读稿
    返回 最终可发送消息
```
