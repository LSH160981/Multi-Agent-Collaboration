# 安装与使用

## 安装

推荐直接安装两个 skill：

- `skills/Multi-Agent-Collaboration/`
- `skills/mac/`

例如：

```bash
mkdir -p ~/.openclaw/workspace/skills
cp -r skills/Multi-Agent-Collaboration ~/.openclaw/workspace/skills/
cp -r skills/mac ~/.openclaw/workspace/skills/
```

然后新开一个 session，或重启/刷新 OpenClaw skills。

---

## 使用方式（最简版）

只记住 3 种：

### 1. 直接说任务
安装后，复杂任务默认就走本 skill。

例如：

```text
帮我调研一下最近值得关注的 OpenClaw 多 Agent 项目，并总结改造建议
```

### 2. 用 `/mac`
这是最直接的强制入口。

```text
/mac 帮我做一个多步骤调研，并输出结构化报告
```

### 3. 直接点名 skill

```text
使用 Multi-Agent-Collaboration skill 完成这个任务
```

---

## 你真正需要理解的就一句话

**安装完后，普通复杂任务默认就归它管；想强制进入多 Agent，就用 `/mac`。**

## 安装后测试

### 测试 1：两两握手

所有 Agent 互相发送：

```text
我是XXagent，你是谁？
```

回复格式：

```text
我是XXagent，我的能力是XXX。
```

### 测试 2：静默任务测试

任务：

```text
搜索 GitHub 最近 7 天 star 涨得最快的 10 个项目，总结共同特点。
```

目标：

- 测试 Agent 是否还能持续工作
- 测试审核链
- 测试主Agent汇总输出是否去重

### 测试 3：原生 session 调度 demo

```bash
./scripts/runtime_sessions.py "/mac 调研最近 30 天值得关注的 OpenClaw 多Agent 项目，并给出改造建议"
```

---

## 注意

- 只有主Agent可以联系用户
- 非主Agent禁止与用户通信
- 所有通信优先 JSON 结构化
- `/mac` 最稳的实现方式是增加一个 `mac` 命令桥 skill
