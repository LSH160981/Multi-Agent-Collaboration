# 安装与使用

## 安装

### 方式 1：安装主 skill

把 `skills/Multi-Agent-Collaboration/` 放到 OpenClaw 工作区的 `skills/` 目录下。

例如：

```bash
mkdir -p ~/.openclaw/workspace/skills
cp -r skills/Multi-Agent-Collaboration ~/.openclaw/workspace/skills/
```

### 方式 2：连同 `/mac` 命令桥一起安装（推荐）

把下面两个 skill 都放到工作区 `skills/` 下：

- `skills/Multi-Agent-Collaboration/`
- `skills/mac/`

例如：

```bash
mkdir -p ~/.openclaw/workspace/skills
cp -r skills/Multi-Agent-Collaboration ~/.openclaw/workspace/skills/
cp -r skills/mac ~/.openclaw/workspace/skills/
```

然后：

- 新开一个 session，或
- 重启 gateway，或
- 让 OpenClaw 刷新 skills

## 使用方式

### 方式 1：直接对话

安装后，遇到复杂任务时，本 skill 的方法论应默认生效。

### 方式 2：显式触发 `/mac`

```text
/mac 帮我做一个多步骤调研，并输出结构化报告
```

如果平台支持原生 skill slash command，`mac` skill 会让 `/mac` 出现在命令里。
如果平台不支持原生 slash command，用户直接发送纯文本 `/mac xxx` 也应视为强制触发。

### 方式 3：明确指定主 skill

```text
使用 Multi-Agent-Collaboration skill 完成这个复杂开发任务
```

## 推荐入口语义

用户安装后，默认就是在和本 skill 的主Agent方法论对话。

也就是说：

- 普通复杂任务 → 默认走本 skill
- `/mac` → 强制走本 skill
- 口头点名本 skill → 强制走本 skill

## 安装后测试

### 测试 1：两两握手

所有 Agent 互相发送：

```text
我是XXagent，你是谁？
```

回复格式：

```text
我是XXagent，我的能力是XXX。
```

### 测试 2：静默任务测试

任务：

```text
搜索 GitHub 最近 7 天 star 涨得最快的 10 个项目，总结共同特点。
```

目标：

- 测试 Agent 是否还能持续工作
- 测试审核链
- 测试主Agent汇总输出是否去重

### 测试 3：原生 session 调度 demo

可运行：

```bash
./scripts/runtime_sessions.py "/mac 调研最近 30 天值得关注的 OpenClaw 多Agent 项目，并给出改造建议"
```

这会：

- 把 `/mac` 解析成任务包
- 生成 A/B 组编组计划
- 以主Agent / AgentPool / 审核Agent / 检查Agent四个角色视角分别发起真实 OpenClaw agent turn
- 把结果写入 `examples/generated/native-sessions/`

## 注意

- 只有主Agent可以联系用户
- 非主Agent禁止与用户通信
- 所有任务要写队列与日志
- 所有通信优先 JSON 结构化
- `/mac` 最稳的实现方式是增加一个名字就是 `mac` 的命令桥 skill，而不是只依赖主 skill 名字
