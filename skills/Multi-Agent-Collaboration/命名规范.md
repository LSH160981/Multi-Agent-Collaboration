# 命名规范

## 总原则

Agent 名字必须让人一眼看懂职责，建议采用：

- 中文主名 + 英文尾缀
- 或 组别 + 职能

例如：

- 主Agent-Main
- 审核Agent-Review
- 检查Agent-Inspector
- AgentPool-HR
- A组-组长-Lead
- A组-前端专家-Frontend
- B组-检索专家-Research
- B组-测试专家-Test

## task_id 规范

建议：

```text
TASK-YYYYMMDD-序号
```

例如：

```text
TASK-20260321-001
```

## group_id 规范

- GROUP-A
- GROUP-B

## 消息 id 规范

```text
MSG-YYYYMMDDHHMMSS-随机短串
```

## 文件命名

- 伪代码：中文名允许保留
- 协议文件：优先英文或中英结合
- 队列文件：`queue/<task_id>.json`
- 日志文件：`logs/YYYY-MM-DD.md`
- 记忆文件：`memory/YYYY-MM-DD.md`
