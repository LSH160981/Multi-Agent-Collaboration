# git 记录策略

目标：记录记忆、日志、关键产物，同时减少硬盘消耗。

## 原则

1. commit message 必须使用中文
2. 只记录关键变更，不提交无意义噪音
3. 大型缓存、临时文件、重复中间产物不入库
4. 日志优先按天滚动，避免无限增长
5. 记忆优先摘要化，避免把整个对话原样写入

## 建议提交内容

- 任务最终产物
- 关键研究结论
- 伪代码/流程更新
- Agent 能力边界调整
- 审核结论
- 重要复盘

## 不建议提交内容

- 大量重复搜索结果
- 临时下载缓存
- 无价值的中间状态截图
- 可重新生成的大文件

## 建议提交频率

- 一个任务一个主提交
- 大改动再拆分为 2-4 个主题提交
- 避免每分钟提交一次

## 建议 commit 格式

- `新增：主Agent任务汇总模板`
- `修复：检查Agent巡检遗漏重试逻辑`
- `优化：多Agent消息去重流程`
- `研究：补充OpenMOSS巡查机制笔记`
