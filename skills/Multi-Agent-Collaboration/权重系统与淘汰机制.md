# 权重系统与淘汰机制

## 目标

让多 Agent 团队不是固定死板结构，而是像公司一样：

- 表现好 → 加权保留
- 表现差 → 降权、淘汰、重招

## 一、评分来源

权重调整主要依据：

1. 审核通过率
2. 总评分
3. 返工次数
4. 是否按时推进
5. 是否经常卡死或失联
6. 是否需要检查Agent频繁唤醒

## 二、基础公式（建议）

```text
综合表现分 = 0.35*通过率 + 0.30*平均审核分 + 0.15*时效分 + 0.10*稳定性分 + 0.10*协作分
```

## 三、权重调整建议

- 综合表现分 >= 85：提升权重，优先复用
- 70 <= 综合表现分 < 85：正常保留
- 60 <= 综合表现分 < 70：降权观察
- 综合表现分 < 60：建议淘汰或重建

## 四、组级淘汰

如果某一组出现以下情况，可直接判定整组重建：

- 连续 2 次审核明显落后另一组
- 多名 specialist 长时间无产出
- 组长汇总能力明显不足
- 需要检查Agent反复强制唤醒仍不推进

## 五、淘汰后的处理

- 保留失败案例用于复盘
- 不保留无意义历史上下文
- 只继承必要的任务背景与失败教训
- 用新 specialist / 新组重新执行

## 六、注意

权重系统是辅助决策，不应机械化地替代主Agent与审核Agent的判断。
