# 测试脚本

## 测试 1：Agent 两两握手

目标：验证所有固定角色与后续新建角色是否具备基本 A2A 通信能力。

### 发送模板

```text
我是XXagent，你是谁？
```

### 回复模板

```text
我是XXagent，我的能力是XXX。
```

### 要求

- 每个 Agent 都要给每个 Agent 发一次
- 必须记录日志
- 必须记录成功/失败次数
- 若失败，检查 JSON 协议、session 路由、日志路径是否正常

## 测试 2：静默任务测试

目标：验证安装后所有 agent 是否还能持续工作，并完成一次完整协作链。

### 任务

```text
搜索 GitHub 最近 7 天 star 涨得最快的 10 个项目，总结共同特点。
```

### 期望

- 主Agent 负责唯一用户出口
- AgentPool 生成对应角色
- A/B 两组并行完成调研
- 审核Agent 评分并择优
- 检查Agent 在执行中持续巡检
- 输出必须去重

## 测试 3：恢复测试

目标：验证 stale / watch agent 能否被检查Agent识别并恢复。

### 要求

- 人工制造一个无动作队列项
- 运行巡检脚本
- 确认产生唤醒或恢复建议
- 必要时执行 retry / 重派 / 重建

## 测试 4：原生 session 调度测试

目标：验证系统不只是生成 JSON，而是真正调用 OpenClaw 原生 session / agent turn。

### 命令

```bash
./scripts/runtime_sessions.py "/mac 调研最近 30 天值得关注的 OpenClaw 多Agent 项目，并给出改造建议"
```

### 检查点

- 是否生成 `task-packet.json`
- 是否生成 `group-plan.json`
- 是否生成 `native-session-results.json`
- 四个角色（主Agent / AgentPool / 审核Agent / 检查Agent）是否都拿到各自视角任务
- 是否能看到真实 OpenClaw agent turn 返回结果
