# 自动自学习方案（最终版）

## 目标

让本 skill 周期性学习新的多 Agent 协作方案，并把真正有价值的部分吸收到自己的：

- 项目骨架
- 伪代码
- 通信协议
- 审核体系
- 恢复体系
- 测试体系

而不是只做“收集链接”。

---

## 学习来源

优先级如下：

1. OpenClaw 官方文档
2. ClawHub 上相关多 Agent skill
3. GitHub 上的多 Agent / orchestration / collaboration 项目
4. 博客文章、架构分享、案例复盘

重点关注：
- OpenMOSS
- OpenClaw 生态内的多会话 skill
- 任务状态机清晰的项目
- 审核 / 恢复机制做得扎实的项目

---

## 每次学习提取什么

每次学习至少提取以下维度：

1. 角色设计
2. 项目骨架
3. 通信协议
4. 任务状态机
5. 审核评分机制
6. 巡检与恢复机制
7. 伪代码与执行流程
8. 自动化测试方法

并回答：
- 对方是怎么拆任务的？
- 对方如何在多 Agent 间通信？
- 对方如何避免噪音？
- 对方如何在失败时恢复？
- 哪些设计能迁移到 OpenClaw 原生 session 体系？

---

## 学习结果落地

学习不是看完就算，而是必须落地到仓库：

- 写入 `research/`
- 更新 `SKILL.md`
- 更新 `伪代码.md`
- 更新 `逻辑执行流程.md`
- 必要时更新 `通信协议.json`
- 必要时更新测试模板与恢复策略

---

## 自学习边界

允许：
- 搜索资料
- 阅读资料
- 提炼优缺点
- 改进本仓库文档与模板
- 用 git 留档

禁止：
- 未经明确要求擅自升级系统
- 未经明确要求修改用户私有配置
- 未经明确要求向外部发送消息
- 把未经验证的外部设计直接强塞进生产流程

---

## 执行频率

建议：
- 每周 1-3 次小规模学习
- 每月 1 次结构化复盘
- 每次重大故障后追加一次针对性学习

---

## 与 git 的结合

每次真正吸收改动后：

1. 先写研究笔记
2. 再改文档或模板
3. 最后 git commit

要求：
- commit message 使用中文
- 尽量做结构化小步提交
- 减少无意义缓存和大文件，节省硬盘

---

## 自学习输出模板

每次学习应至少产出：

1. 来源
2. 摘要
3. 项目骨架提炼
4. 伪代码提炼
5. 通信逻辑提炼
6. 审核逻辑提炼
7. 恢复逻辑提炼
8. 是否值得迁移
9. 如果迁移，具体改哪些文件

这样它才是真正的“自我进化”，而不是“自动收藏夹”。
