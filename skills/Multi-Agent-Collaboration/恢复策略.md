# 恢复策略

## 1. Agent 中断

恢复顺序：

1. 读 `queue/`
2. 读 `logs/`
3. 读最近产物
4. 判断最后成功步骤
5. 从最近成功步骤继续，而不是直接从头重做

## 2. Specialist 挂掉

- 若上下文可恢复：原 Agent 继续
- 若上下文损坏：新建同角色 Agent 接管
- 必要时只继承任务上下文，不继承全部历史对话

## 3. Lead 挂掉

- 从组内 specialist 的队列和日志恢复进度
- 临时指定新的组长
- 汇总未交付项后继续

## 4. Group 整体失败

- 审核Agent记录失败原因
- AgentPool 解散低效组
- 重新招聘替代组
- 保留旧组失败案例供后续评分与学习

## 5. Review 中断

- 保留已完成评分项
- 从未完成项恢复
- 已出结论的条目不得重复评分造成结果漂移

## 6. Inspection 中断

- 优先恢复检查Agent
- 因为检查Agent负责系统活性与强制执行

## 7. 主Agent中断

恢复时必须先查看：

- 已给用户发过什么
- 当前任务最新状态
- 审核通过的最终结果

防止恢复后重复回复用户。
