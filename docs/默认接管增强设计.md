# 默认接管增强设计

## 目标

在不强改 OpenClaw 内核的前提下，让用户安装本 skill 后，复杂任务默认落入 Multi-Agent-Collaboration 流程。

## 分层策略

### 第 1 层：方法论接管

- 在 SKILL.md 中明确：复杂任务默认走多 Agent 流程
- 把 `/mac` 作为显式入口
- 把“使用 Multi-Agent-Collaboration skill”作为明确入口

### 第 2 层：共享安装接管

- 安装到 `~/.openclaw/skills/`
- 让所有 Agent 默认都能读到本 skill

### 第 3 层：初始化接管

- 安装后自动初始化 `mac-system/`
- 预生成主Agent / 审核Agent / 检查Agent / AgentPool 骨架
- 提示执行握手测试与静默测试

### 第 4 层：对话策略接管

在复杂任务场景下，主Agent默认先判断：

- 是否需要拆解
- 是否需要招聘 specialist
- 是否需要双组竞争
- 是否需要审核与巡检

而不是立即单 Agent 自行处理全部任务。

## 为什么暂不做更深内核改动

- 更深集成通常需要依赖 OpenClaw 插件、配置或核心行为的更强联动
- 当前最稳路线是先把方法论、目录结构、模板、脚本、协议、样例全做完整
- 这样更利于后续升级到更深集成模式
