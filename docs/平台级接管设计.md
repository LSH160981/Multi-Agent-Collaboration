# 平台级接管设计

目标：用户安装本 skill 后，默认就像是在和 Multi-Agent-Collaboration 的主Agent方法论对话。

## 接管思路

### 1. 方法论接管

最现实、最稳的第一步不是强改 OpenClaw 内核，而是：

- 让 skill 文档明确要求复杂任务默认进入多 Agent 流程
- 让 `/mac` 成为显式入口
- 让主Agent方法论成为默认复杂任务处理策略

### 2. 共享技能安装

将 skill 安装到：

- `~/.openclaw/skills/Multi-Agent-Collaboration`

这样所有 Agent 都可见。

### 3. 主Agent 中心化

用户真正看到的只有主Agent。
其他 Agent 只做内部协作。

### 4. 多平台兼容

对 TUI / GUI / TG 的一致性，不强依赖平台原生 slash command 是否存在。
而是：

- 真有 slash command 时，用 `/mac`
- 没有时，把用户发出的 `/mac xxx` 纯文本也视为触发词
